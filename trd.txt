# 📊 Document Overview

This TRD defines the technical architecture and implementation requirements for DB-Lighthouse AI — a web-based PostgreSQL optimization and visualization tool.

Design Goals:

* Easy to build with AI coding tools (Cursor, Claude, v0.dev)
* Docker-based backend for DB analysis
* Firebase for auth + metadata
* Fully local LLM integration (Jan AI)
* Zero external paid API dependency
* Works with DB sizes up to 5GB
* Graceful fallback if AI fails

---

# 🏗️ System Architecture

```
Client (Next.js 15 App Router)
        |
        v
Next.js API Routes (Node runtime)
        |
        v
Python FastAPI Service (Docker)
        |
        +----> Shadow PostgreSQL (Docker)
        |
        +----> User Connected PostgreSQL (Read-only)
        |
        +----> Local LLM (Jan AI - OpenAI Compatible Endpoint)
        |
        v
Firebase (Auth + Metadata Storage)
```

### Responsibilities

| Layer      | Responsibility                  |
| ---------- | ------------------------------- |
| Next.js    | UI, OAuth, Graph rendering      |
| API Routes | Orchestration                   |
| FastAPI    | DB scanning, heuristics         |
| Shadow DB  | Safe testing                    |
| Firebase   | Users + scan history            |
| Jan AI     | Schema reasoning + explanations |

# 🔌 API Design

All endpoints under `/api/`

---

### POST `/api/connect-db`

**Purpose:** Connect to PostgreSQL and create shadow clone

Input:

```
{
  connection_string: string
}
```

Output:

```
{
  success: boolean,
  table_count: number
}
```

---

### POST `/api/run-storage-scan`

Purpose: Run Space Shrinker analysis

Input:

```
{
  project_id: string
}
```

Output:

```
{
  scan_id: string,
  storage_before_mb: number,
  storage_after_mb: number,
  recommendations: []
}
```

---

### POST `/api/run-index-analysis`

Purpose: Suggest indexes

Input:

```
{
  project_id: string
}
```

Output:

```
{
  recommendations: []
}
```

---

### POST `/api/apply-to-shadow`

Purpose: Apply SQL patch safely

Input:

```
{
  project_id: string,
  sql_patch: string
}
```

Output:

```
{
  success: boolean,
  new_storage_estimate: number
}
```

---

### POST `/api/generate-schema`

Purpose: Natural language → SQL

Input:

```
{
  description: string
}
```

Output:

```
{
  sql_schema: string,
  tables: []
}
```

---

### POST `/api/github/import`

Input:

```
{
  repo_url: string,
  access_token?: string
}
```

Output:

```
{
  sql_files: []
}
```

# 🔒 Security & Rate Limiting

### Security Rules

* DB credentials never stored in Firestore
* Stored only in FastAPI memory
* Shadow DB isolated per session
* All SQL patches validated using regex whitelist:

  * Allowed: ALTER TABLE, CREATE INDEX
  * Blocked: DROP DATABASE, DROP TABLE

### Rate Limits

Using `express-rate-limit` (Next.js middleware)

* 10 scans per hour per user
* 50 API requests per hour per user

# 🤖 AI Integration

### Endpoint

Jan AI local endpoint:

```
http://localhost:1337/v1/chat/completions
```

### Model

User-configurable model (default: llama3)

### AI Used For:

* Explaining recommendations
* Ranking impact
* Generating schema from description

### Prompt Template:

System:
"You are a PostgreSQL optimization expert. Only output structured JSON."

Fallback Mode:

If AI fails:

* Show heuristic recommendation without explanation
* Mark explanation = "AI unavailable"

# 🚀 Deployment Strategy

### Local Dev Setup

1. Clone repo
2. Run `docker-compose up`
3. Run `npm install`
4. Run `npm run dev`

### Docker Compose Services

* nextjs
* fastapi
* shadow-postgres
* jan-ai (external, optional)

### Production (Later)

* Deploy Next.js on Vercel
* Deploy FastAPI on Render
* Replace local Jan AI with hosted LLM

# 🎯 Technical Success Criteria

System is “Done” when:

* Can connect to 1–5GB PostgreSQL database
* Can generate ≥5 meaningful optimization suggestions
* Can show measurable storage reduction %
* Can suggest at least 1 missing index on test DB
* Graph visualization works with 20+ tables
* Full system runs with single `docker-compose up`
* LLM failure does not crash system

# Final Architecture Philosophy

* Heuristics first
* AI second
* Safe shadow always
* Zero destructive production writes
* Buildable by solo dev in 3–4 weeks
* Clean GitHub portfolio showcase
