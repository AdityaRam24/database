Schema canvas & node cards
Critical
Node cards are too dark and low-contrast
The near-black card background with small white text violates WCAG AA. Table names, row counts, and size labels are all competing at similar visual weight — nothing has hierarchy.
Now
bg ~#1a1a2e · all text same weight · no scan path
Fix
White/light card surface · table name 500 weight 14px · metadata muted 11px below
Critical
Row count & size info is illegible at small scale
The "4715 ROWS / 440.0 B" metadata renders at ~10px and is barely readable. Data density is wasted because users can't skim it.
→ Minimum 12px for any data label. Use two tiers: table name at 13px/500, metadata at 11px/muted gray.
Medium
No visual weight difference between table types
Junction tables (playlisttrack, invoiceline), entity tables (customer, artist), and lookup tables (genre, mediatype) all look identical. There's no color-coding or shape signal to tell them apart.
→ Introduce subtle left-border accent colors: purple for entities, teal for junction/relation tables, amber for lookup/reference tables.
Medium
Foreign key edges are invisible against the dark background
The dashed relationship lines are 0.5–1px and nearly the same color as the canvas. Users can't follow relationships without zooming in significantly.
→ Increase edge opacity, use color-coded edges (same hue as the connected table category), add subtle arrowhead direction cues.
Quick win
No hover state on nodes
Hovering a node should highlight all its connected edges and dim everything else. Right now, nothing responds on hover — the canvas feels static and dead.
→ On hover: elevate the node + its edges to full opacity, reduce all unrelated nodes to 25% opacity.
Quick win
Canvas background is too neutral — no depth cue
The flat gray canvas and the dark cards have very low contrast between them. The schema "floats" but doesn't feel grounded.
→ Subtle dot-grid or cross-hatch canvas pattern (2px dots at 24px spacing, 8% opacity). Standard in draw.io, Figma, Miro.
Navigation & toolbar
Critical
Search bar does not show column-level results
The "Find table or column…" input exists but there's no visible feedback about what it's searching, and no autocomplete preview. Users searching for "customer_id" don't know if it will match column names or just table names.
Now
Plain text input, no feedback, no results preview
Fix
Typeahead dropdown: table results grouped above column results · jump-to on click
Critical
Heatmap button is isolated and unexplained
The "Heatmap" button sits alone in the top-right with no indication of what it does, what metric it colors, or how to read it. New users will not click it.
→ Add a tooltip on hover: "Color tables by row count" or "Color by storage size." When active, show the active legend inline in the top bar.
Medium
"All Tables" filter chip has no alternatives visible
The "All Tables" pill implies there are other filter states, but users can't see them without clicking. This hides important navigation.
→ Show chips for each table category inline: "All · Entities · Relations · Lookups" so users can filter by type at a glance.
Medium
No zoom controls visible
"Scroll to zoom · Drag to pan" is discoverable text but there are no visual controls. Users on trackpads with non-standard scroll behavior won't know they're stuck.
→ Add a small zoom control cluster: [−] [fit to screen] [+] in the bottom-right corner of the canvas.
Optimization report panel
Medium
"All Good!" message is a dead end
When there are no issues, the panel shows a success state but offers nothing to do next. Users who open this section want to learn — give them something.
→ Below the success message, add: schema stats summary (total tables, total relations, heaviest table), and a "Run deep analysis" CTA for index coverage, missing FK constraints, etc.
Quick win
Report section lacks timestamp
Users can't tell when this optimization check was last run. Is it live? Cached? From yesterday?
→ Add "Last checked: 2 minutes ago" with a manual refresh icon next to the "Optimization Report" heading.
Left sidebar & workspace
Medium
Sidebar has no table list — forces canvas-only navigation
The sidebar shows only the database name "db1" and "+ Connect New Database." There's no list of tables, no recent queries, no favorites. For schemas with 20+ tables, canvas navigation alone is exhausting.
→ Add a collapsible table list in the sidebar with search + scroll. Click to focus the canvas on that node.
Quick win
Workspace name "Steve Team" has no context
The workspace footer shows name + "WORKSPACE" label but no connection status indicator, no environment tag (prod/dev/staging).
→ Add a small colored dot: green = connected, yellow = connecting, red = disconnected. Add an env badge if multiple DBs are supported.
Quick win
Legend is placed at bottom-left but rarely noticed
The Focal Node / Search Match / High-Degree Node legend is buried at the bottom of the canvas. Users encounter these states before reading the legend.
→ Move legend to a floating pill in the top-left of the canvas, or collapse it into a "?" icon that expands on hover.
